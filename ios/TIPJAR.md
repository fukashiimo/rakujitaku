# 投げ銭（コーヒーをおごる）リリース手順

CLEAR画面に「☕️ 開発者にコーヒーをおごる」ボタンを実装済み。
**App Store Connectに課金商品が未設定の間、ボタンは自動的に非表示**になるため、
このコードが入ったビルドをいつ出しても安全。

実装の検証状況:
- ユニットテスト（StoreKitTest）で「商品読み込み→購入→サンクス表示」を自動検証済み
- Xcodeで手動確認する場合: Runで起動（スキームにStoreKitテスト構成設定済み）→
  支度を完了してCLEAR画面へ → コーヒーボタンが表示され、テスト購入できる

## リリースまでの手順（v1.1向け）

### 1. 有料App契約を結ぶ（最初にやる・承認に数日かかることがある）

1. https://appstoreconnect.apple.com/ → **ビジネス** → **契約**
2. 「**有料App**（Paid Apps）」の契約に同意
3. **銀行口座**（受取先）と**税務情報**（日本の場合は米国税務フォーム W-8BEN を画面の案内どおり入力）を登録
4. Statusが **Active（有効）** になるまで待つ ← これが済むまで課金商品は審査に出せない

### 2. 課金商品を作成

1. App Store Connect → らくじたく → **収益化 → App内課金**（In-App Purchases）→「+」
2. 入力内容:
   - **タイプ**: 消耗型（Consumable）
   - **参照名**: Coffee Tip
   - **製品ID**: `com.shuyafukai.rakujitaku.tip.coffee` ← コードと完全一致させること
3. **価格**: ¥300（好みで¥100〜500に変更可。コード側の修正は不要）
4. **ローカリゼーション（日本語）**:
   - 表示名: コーヒー1杯
   - 説明: 開発を応援していただきありがとうございます！
5. **審査用スクリーンショット**: CLEAR画面（ボタンが写っているもの）をアップ
   ※シミュレータをStoreKit構成付きで起動して撮影する（Xcode Run → ⌘S）

### 3. バージョン1.1として提出

1. `ios/project.yml` の `MARKETING_VERSION` を `1.1.0` に上げ、`xcodegen generate`
2. Archive → Upload（1.0と同じ手順）
3. App Store Connectで新バージョン「1.1.0」を作成し、ビルドを紐付け
4. **バージョンページ下部の「App内課金」欄で作成した商品を選択**（初回はアプリ本体と一緒に審査される）
5. 審査へ提出

## 注意

- 製品IDは一度使うと削除後も再利用できない。タイプミスに注意
- 投げ銭は「消耗型」なので何度でも購入できる（リストア不要）
- 価格を変えたくなったらApp Store Connect側の変更だけでOK（コード側は表示価格を自動取得）
